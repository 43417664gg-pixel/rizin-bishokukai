## 最重要の前提

### 1. `site/js/demo-data.js` が正本
Supabaseは**同期先**であって正本ではない。データを変えるときは必ず demo-data.js を編集し、下記の同期ボタンで反映する。

### 2. 反映は admin の同期ボタンから
1. `admin.html` を開く → Gakuのメール＋パスワードでログイン
2. **必ずページをリロードしてから**（古いJSが動いて反映されない事故が実際に起きた）
3. 「⚡ ローカルデータをSupabaseへ同期」を押す
4. 結果表示に「fighters: 40件」等が出れば成功

> Supabaseダッシュボード（SQL Editor）は**Chrome翻訳干渉＋高負荷で頻繁に無反応**になる。150秒待っても真っ黒のことがある。DDL以外はこのボタンで済ませる。

### 3. 決定打ルール（★このルールの正本はここ）
「あらます」の判定は**公式の決着表記ではなく、相手を倒した一撃**で行う。

> 右フックでダウン → パウンドでストップ　＝　決定打は **右フック**

MMAの公式表記は「TKO（パウンド）」が大半で、公式表記に合わせると`パウンド`ばかりが正解になり「あの右フックで倒れる」という読みが死ぬため。`パウンド（削り切り）`は殴り続けて削り切った試合だけに使う。
※ブックメーカーはKO/TKO・一本・判定までしか市場を作らない（技名は裁定が揉めるため）。**仲間内だから裁定できる**という前提の上に成り立つルール。

## ルール（三段梯子）
1. **的中** — 勝者を当てる。ランキングの主役。分母は「結果が出た予想試合」だけ
2. **ピタリ** — 決まり手＋ラウンドまで一致（判定決着は決まり手のみでピタリ）
3. **あらます** — 決定打まで完全一致した最上位

勝者を外したら以降すべて不成立。ドロー・無効試合は集計から除外。
